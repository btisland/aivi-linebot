# 產品上下文

## 產品定位

### 核心價值主張
AIVI News LINE Bot 是一個**輕量級的新聞推播工具**，讓使用者透過簡單的指令即時取得 AIVI 科技博客的最新文章。

**關鍵特色**:
- 📰 **即時更新**: 直接爬取最新文章，無需預先同步
- ⚡ **快速回應**: 10 秒內完成回覆
- 🤖 **零學習成本**: 單一指令 `/aivi` 即可使用
- 🛡️ **可靠穩定**: 完整錯誤處理，友善錯誤訊息

## 目標使用者

### 主要使用者群
1. **AIVI 科技博客讀者**
   - 需要快速瀏覽最新文章
   - 透過 LINE 作為主要通訊工具
   - 偏好即時推播而非定期檢查網站

2. **技術愛好者**
   - 關注 AI/科技趨勢
   - 習慣使用聊天機器人取得資訊
   - 重視資訊取得效率

### 使用者需求
- ✅ 快速取得最新文章標題和連結
- ✅ 不需要開啟瀏覽器
- ✅ 在 LINE 中一站式完成資訊瀏覽
- ❌ 不需要複雜的設定或訂閱流程

## 核心功能

### 1. 新聞查詢指令
**指令**: `/aivi`

**功能描述**:
- 爬取 AIVI 科技博客最新 5 則文章
- 回傳標題和連結
- 不區分大小寫（`/aivi`, `/AIVI`, `/Aivi` 都可）

**使用流程**:
```
使用者: /aivi
    ↓
Bot: 🔍 正在搜尋最新文章...
    ↓
Bot:
📰 AIVI 最新文章

1. [文章標題]
   🔗 https://...

2. [文章標題]
   🔗 https://...

...
```

**回應時間目標**: < 10 秒

### 2. 錯誤處理
**網路錯誤**:
```
抱歉，目前無法取得新聞資訊。
請稍後再試，或直接訪問 AIVI 博客。
```

**無法解析**:
```
無法解析新聞內容，請稍後再試。
```

### 3. 指令不匹配
**非指令訊息**: 忽略（不回應）
**未知指令**: 忽略（未來可加入提示）

## 產品限制

### 目前不支援的功能
- ❌ 定期自動推播
- ❌ 使用者訂閱管理
- ❌ 文章全文預覽
- ❌ 文章分類/標籤篩選
- ❌ 歷史文章搜尋
- ❌ 多個新聞來源

### 設計取捨
**為什麼不支援定期推播？**
- 需要儲存使用者訂閱資料（資料庫）
- 需要排程系統（cron job）
- 增加系統複雜度
- 目前定位：按需查詢，保持簡單

**為什麼只支援 AIVI？**
- 專注單一資料源，確保品質
- 爬蟲邏輯清晰，易於維護
- 未來可擴展（模組化設計已預留空間）

## 使用者回饋考量

### 預期問題
1. **「為什麼只有 5 則文章？」**
   - 回應: 平衡資訊量和閱讀體驗，5 則是最佳數量
   - 未來: 可支援自訂數量 `/aivi 10`

2. **「可以定期推播嗎？」**
   - 回應: 目前支援按需查詢，定期推播規劃中
   - 未來: Phase 3 功能

3. **「能支援其他新聞來源嗎？」**
   - 回應: 已規劃，將逐步新增
   - 未來: Phase 2 功能

### 改進方向
- 根據使用頻率調整文章數量
- 監測回應時間，優化爬蟲效能
- 收集使用者偏好，規劃新功能
