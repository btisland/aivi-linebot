# 專案簡介

## 專案概述

**專案名稱**: AIVI News LINE Bot
**類型**: LINE Messaging API 聊天機器人
**階段**: MVP 已完成，生產環境運行中
**授權**: MIT License

### 一句話描述
輕量級的 LINE Bot，當使用者傳送 `/aivi` 指令時，即時爬取 AIVI 科技博客最新 5 則文章並回傳。

## 專案目標

### 主要目標
1. **提供便捷的資訊取得方式**
   - 使用者無需開啟瀏覽器即可查看最新文章
   - 透過 LINE 這個日常通訊工具完成資訊瀏覽

2. **展示技術能力**
   - 整合 LINE Messaging API
   - 實作輕量級網頁爬蟲
   - 自動化 CI/CD 流程
   - 容器化部署

3. **建立可擴展的基礎架構**
   - 模組化設計，易於新增功能
   - 完整的測試覆蓋
   - 標準化的開發流程

### 成功標準
- ✅ 回應時間 < 10 秒
- ✅ 測試覆蓋率 > 80%
- ✅ 自動化部署流程
- ✅ 穩定運行（錯誤率 < 5%）
- ✅ 程式碼品質（型別提示、文件完整）

## 專案範圍

### 包含的功能 ✅

## 專案背景

### 為什麼建立這個專案？
1. **學習目標**
   - 實踐 LINE Messaging API 整合
   - 探索現代 Python 生態系（httpx, selectolax, uv）
   - 體驗 Hugging Face Space 部署

2. **實際需求**
   - 提供便捷的 AIVI 文章查詢方式
   - 在 LINE 中一站式完成資訊瀏覽
   - 避免每次都要開啟瀏覽器

3. **技術實驗**
   - 測試輕量級爬蟲方案（selectolax vs BeautifulSoup）
   - 實踐非同步程式設計（httpx async）
   - 建立標準化的專案結構和開發流程

### 技術選型考量
- **LINE Bot**: 使用者基數大，整合容易
- **Python**: 豐富的生態系，開發效率高
- **Hugging Face Space**: 免費、支援 Docker、部署簡單
- **uv**: 快速的套件管理，改善開發體驗

## 專案制約

### 技術限制
- LINE Reply Message API 限制（每個事件只能回覆一次）
- Flask 同步框架（在 async scraper 中需使用 `asyncio.run()`）
- Hugging Face 免費方案的資源限制

### 時間限制
- 個人專案，利用業餘時間開發
- MVP 目標：2 週內完成基礎功能

### 預算限制
- 零預算專案
- 僅使用免費方案（GitHub, HF Space, LINE API）

## 風險評估

### 高風險項目
1. **AIVI 網站結構變更**
   - 影響: 爬蟲失效
   - 緩解: 增加錯誤處理，定期檢查

2. **LINE API 政策變更**
   - 影響: 功能受限或成本增加
   - 緩解: 關注官方公告，評估替代方案

### 中風險項目
1. **Hugging Face Space 穩定性**
   - 影響: 服務中斷
   - 緩解: 設定監控，準備備援方案（如 Railway, Render）

2. **流量突然增加**
   - 影響: 超出免費方案限制
   - 緩解: 監控使用量，必要時升級方案

### 低風險項目
1. **相依套件漏洞**
   - 緩解: 使用 `uv lock` 鎖定版本，定期更新

2. **測試覆蓋不足**
   - 緩解: 持續監控覆蓋率，保持 > 80%
